# AI 驱动浏览器诊断方案

> 用 Playwright 获取摄像机网页 DOM，交给 AI 模型分析和操作，替代硬编码选择器和视觉模型截图。

## 背景

当前浏览器诊断有两个问题：
1. 截图模式依赖视觉模型（如 qwen-vl-plus），deepseek-chat 等文本模型不支持
2. DOM 模式用硬编码选择器猜登录表单，不同厂商摄像机（海康、大华、宇视等）HTML 结构各不相同，猜不中就登录失败

## 核心思路

**Playwright 负责获取 DOM + 执行操作，AI 模型负责理解 DOM + 给出指令**

```
Playwright 获取 HTML → 发给 AI 分析 → AI 返回选择器/结论 → Playwright 执行
```

## 三步流程

### 第一步：AI 辅助登录

1. Playwright 打开摄像机网页，获取登录页 HTML（只取 form 和 input 相关元素，控制 token 量）
2. 发给 AI 模型：

```
以下是一台摄像机登录页面的 HTML 片段：
{html}

请分析这个登录页面，返回 JSON 格式：
{
  "usernameSelector": "用户名输入框的 CSS 选择器",
  "passwordSelector": "密码输入框的 CSS 选择器",
  "submitSelector": "登录按钮的 CSS 选择器"
}

只返回 JSON，不要其他内容。
```

3. AI 返回选择器 JSON
4. Playwright 用选择器填入用户名密码，点击登录
5. 等待页面跳转

### 第二步：AI 辅助导航到国标配置页

1. 登录成功后，Playwright 获取页面菜单/导航区域的 HTML
2. 发给 AI 模型：

```
以下是摄像机管理页面的导航菜单 HTML：
{html}

请找到与 GB28181、国标、SIP、平台接入相关的配置页面链接。
返回 JSON 格式：
{
  "selector": "目标链接的 CSS 选择器",
  "description": "这个链接是什么"
}

如果需要多级导航（先点一级菜单再点二级），返回数组：
[
  { "selector": "一级菜单选择器", "description": "网络配置" },
  { "selector": "二级菜单选择器", "description": "GB28181" }
]

只返回 JSON，不要其他内容。
```

3. Playwright 按顺序点击导航

### 第三步：AI 提取配置并对比

1. 导航到国标配置页后，Playwright 获取页面表单 HTML
2. 连同数据库中的期望配置一起发给 AI：

```
以下是摄像机 GB28181 配置页面的 HTML：
{html}

我们平台数据库中该设备的期望配置：
- SIP 服务器 IP: {expectedSipIp}
- SIP 服务器端口: {expectedSipPort}
- 服务器编码(Server ID): {expectedServerId}
- SIP 域(Realm): {expectedRealm}
- 设备编码: {deviceId}

请：
1. 从 HTML 中提取摄像机当前的国标配置值
2. 逐项与期望配置对比
3. 指出不一致的项，并给出修改建议

返回格式：
{
  "extractedConfig": {
    "sipServerIp": "实际值",
    "sipServerPort": "实际值",
    "serverId": "实际值",
    "realm": "实际值",
    "deviceId": "实际值"
  },
  "comparison": [
    { "item": "SIP服务器IP", "expected": "期望值", "actual": "实际值", "match": true/false }
  ],
  "conclusion": "总结：哪些配置正确，哪些需要修改",
  "suggestions": ["建议1", "建议2"]
}
```

## 配置项

在 `appsettings.json` 中：

```json
{
  "Diagnostic": {
    "BrowserCheckMode": "ai-dom",
    // 可选值：
    // "ai-dom"     - AI 分析 DOM（推荐，默认）
    // "dom"        - 硬编码选择器分析 DOM（不依赖 AI，兼容性差）
    // "screenshot" - 截图 + 视觉模型分析（需要视觉模型）
  }
}
```

## 优势

| 对比项 | 硬编码 DOM | 截图+视觉模型 | AI 分析 DOM |
|--------|-----------|-------------|------------|
| 厂商兼容性 | 差，每个厂商要写选择器 | 好，但依赖视觉模型 | 好，AI 能理解任何 HTML |
| 模型要求 | 不需要模型 | 需要视觉模型 | 文本模型即可 |
| 准确性 | 选择器匹配就准 | 取决于截图质量和模型能力 | 高，DOM 信息完整 |
| Token 消耗 | 无 | 高（图片 base64） | 中（HTML 片段） |
| 登录支持 | 差 | 不支持 | 好，AI 能识别任何登录表单 |

## HTML 提取策略（控制 Token）

不发送整个页面 HTML，只提取关键部分：

```csharp
// 登录页：只取 form 和 input 元素
var loginHtml = await page.EvaluateAsync<string>(@"
    () => {
        const forms = document.querySelectorAll('form, .login, [class*=login]');
        if (forms.length > 0) return forms[0].outerHTML;
        // 没有 form 标签，取所有 input
        const inputs = document.querySelectorAll('input, button, [type=submit]');
        return Array.from(inputs).map(el => el.outerHTML).join('\n');
    }
");

// 导航菜单：只取 nav、menu、sidebar 区域
var menuHtml = await page.EvaluateAsync<string>(@"
    () => {
        const nav = document.querySelector('nav, .menu, .sidebar, [class*=menu], [class*=nav]');
        return nav ? nav.outerHTML : document.body.innerHTML.substring(0, 5000);
    }
");

// 配置页：只取 form 和 table 区域
var configHtml = await page.EvaluateAsync<string>(@"
    () => {
        const form = document.querySelector('form, table, .config, [class*=config], [class*=setting]');
        return form ? form.outerHTML : document.body.innerHTML.substring(0, 8000);
    }
");
```
