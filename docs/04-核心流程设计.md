## 1. 设备注册流程

```
摄像机                          SIP Server (.NET)                    MySQL
  │                                 │                                  │
  │──REGISTER(无Auth)──────────────>│                                  │
  │                                 │                                  │
  │<─401 Unauthorized(nonce)────────│                                  │
  │                                 │                                  │
  │──REGISTER(Digest Auth)─────────>│                                  │
  │                                 │──验证密码──>                      │
  │                                 │<─验证通过──                       │
  │                                 │                                  │
  │                                 │──INSERT/UPDATE device────────────>│
  │                                 │                                  │
  │                                 │──SignalR 推送"设备上线"──> 前端    │
  │                                 │                                  │
  │<─200 OK(Expires=3600)───────────│                                  │
  │                                 │                                  │
  │──Keepalive(每60s)──────────────>│──更新心跳时间──────────────────────>│
```

### 关键点
- 首次 REGISTER 无认证头 → 返回 401 + nonce
- 二次 REGISTER 携带 Digest Auth → 验证通过返回 200 OK
- Expires=0 表示注销请求
- 注册成功后更新数据库设备状态为 Online
- 通过 SignalR 推送状态变更到前端

## 2. 视频预览流程

```
前端 (Vue)          .NET API           ZLMediaKit          摄像机
  │                    │                   │                  │
  │─POST /api/stream/play─>│               │                  │
  │  {deviceId,channelId}  │               │                  │
  │                    │                   │                  │
  │                    │─openRtpServer────>│                  │
  │                    │<─返回RTP端口──────│                  │
  │                    │                   │                  │
  │                    │─────────INVITE(SDP: 媒体地址=ZLM)───>│
  │                    │<────────200 OK(SDP)──────────────────│
  │                    │─────────ACK─────────────────────────>│
  │                    │                   │                  │
  │                    │                   │<──RTP PS流────────│
  │                    │                   │  (自动解封装)      │
  │                    │                   │                  │
  │<─返回WebRTC播放地址─│                   │                  │
  │                    │                   │                  │
  │────WebRTC WHEP SDP交换────────────────>│                  │
  │<───WebRTC 视频流─────────────────────── │                  │
```

### 关键点
- .NET 先向 ZLMediaKit 申请 RTP 收流端口
- INVITE 的 SDP 中，媒体地址指向 ZLMediaKit 的 RTP 端口
- 摄像机向 ZLMediaKit 推 PS 流，ZLMediaKit 自动解封装
- 前端通过 WebRTC WHEP 从 ZLMediaKit 拉流播放

## 3. 离线诊断流程

```
DeviceMonitorService (后台定时任务，每30s检查一次)
  │
  │── 扫描所有在线设备
  │── 检查心跳超时 (默认 300s 无心跳)
  │
  ├─ 超时设备:
  │    │
  │    │── 标记设备状态 → Offline
  │    │── SignalR 推送 "设备离线"
  │    │── 创建诊断任务 → diagnostic_tasks 表
  │    │
  │    │── Step 1: ICMP Ping
  │    │    ├─ 成功: "Ping 可达, RTT=2ms" → diagnostic_logs
  │    │    └─ 失败: "Ping 超时" → diagnostic_logs
  │    │
  │    │── Step 2: TCP 端口检查
  │    │    ├─ 检查端口 80, 443, 自定义 WebPort
  │    │    ├─ 成功: "端口 80: 开放" → diagnostic_logs
  │    │    └─ 失败: "端口 80: 不可达" → diagnostic_logs
  │    │
  │    │── Step 3: 浏览器检查 (仅在 Web 端口通时执行)
  │    │    │── Playwright 打开 http://{ip}:{port}
  │    │    │── 尝试登录摄像机网页
  │    │    │── 导航到国标配置页面
  │    │    │── 截图 + 提取 GB28181 配置信息
  │    │    │── 对比平台配置 → 找出差异
  │    │    └── 记录结果 → diagnostic_logs
  │    │
  │    └── 生成诊断结论 → 更新 diagnostic_tasks.conclusion
```

### 浏览器检查的多品牌适配策略
- 不同品牌摄像机网页结构不同
- 可利用 Qwen 视觉模型分析截图，识别配置字段位置
- 也可按品牌维护 Playwright 脚本库

## 4. AI 问答流程

```
用户: "为什么摄像机 34020000001320000001 离线了？"
  │
  ▼
┌───────────────────────────────────────────────────┐
│ AI Agent Service                                   │
│                                                    │
│  1. 构建 messages:                                  │
│     - System Prompt (角色、能力描述)                  │
│     - User Message (用户问题)                        │
│     - Functions 定义列表                             │
│                                                    │
│  2. 发送到 Qwen 3.5 API                             │
│                                                    │
│  3. Qwen 返回 function_call:                        │
│     → get_device_status(deviceId="3402...")          │
│     执行 → 返回 {status: "Offline", lastKeepalive}   │
│                                                    │
│  4. Qwen 再次 function_call:                        │
│     → get_diagnostic_logs(deviceId="3402...")        │
│     执行 → 返回诊断日志列表                           │
│                                                    │
│  5. Qwen 综合分析，生成自然语言回答                    │
└───────────────────────────────────────────────────┘
  │
  ▼
回答: "该摄像机从 2026-03-28 14:30 开始离线。
      诊断结果显示：Ping 可达(RTT=2ms)，Web 80端口开放，
      浏览器检查发现国标配置中 SIP 服务器 IP 配置错误
      (配置为 192.168.1.100，应为 188.18.35.225)。
      建议修正摄像机国标配置中的 SIP 服务器 IP 地址。"
```

### AI Function Calling 定义

| 函数名 | 描述 | 参数 |
|--------|------|------|
| `get_device_status` | 查询设备状态、最后注册/心跳时间 | deviceId |
| `get_diagnostic_logs` | 查询最近诊断日志 | deviceId, limit? |
| `get_device_info` | 查询设备详细信息 | deviceId |
| `run_diagnostic` | 立即执行一次诊断 | deviceId |
| `list_offline_devices` | 列出所有离线设备 | — |
| `get_alarm_records` | 查询设备报警记录 | deviceId, hours? |
