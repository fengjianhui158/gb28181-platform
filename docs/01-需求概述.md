# GB28181 智能视频管理平台 — 需求概述

## 项目背景

轨道交通视频监控项目需要一个基于 Web 的 GB28181 管理平台，用于统一管理摄像机设备，并具备智能诊断和 AI 问答能力。

## 核心功能需求

### 1. GB28181 设备注册管理
- 摄像机通过 GB28181/SIP 协议注册到平台
- 平台作为 SIP 服务器（UAS）接收设备 REGISTER
- 支持 SIP Digest 认证
- 设备目录查询（Catalog）

### 2. Web 视频预览
- 在 Vue 3 网页上实时预览摄像机视频
- 使用 WebRTC 协议，延迟 < 500ms
- 通过 ZLMediaKit 做 RTP → WebRTC 转发

### 3. 设备状态监控
- 实时检测摄像机在线/离线状态
- 基于 SIP 心跳（Keepalive）超时判断
- 通过 SignalR 实时推送状态变更到前端

### 4. 离线自动诊断
摄像机离线后，平台自动执行以下诊断步骤：
- **Step 1**: Ping 摄像机 IP，检查网络连通性
- **Step 2**: 检查摄像机 Web 端口（80/443）是否可达
- **Step 3**: 如果 Web 端口通，使用 Agent Browser 访问摄像机网页，检查国标配置是否正确
- 全过程记录诊断日志

### 5. AI 问答 Agent
- 用户可询问"为什么这台摄像机离线了"
- 大模型基于诊断日志，分析并回答离线原因
- 给出具体的修复建议

## 非功能需求

| 需求 | 描述 |
|------|------|
| 规模 | 100-500 路摄像机 |
| 品牌 | 多品牌混合（海康、大华、宇视、华为等） |
| 延迟 | 视频预览 < 500ms |
| 可用性 | 平台 7×24 运行 |
